The framework · built with Claude

Five layers, designed by Claude, run as one machine.

Claude didn’t just power the agents — it architected the whole framework: the scheduler, the runner, the guards, and the dashboard a human approves from. Hover a layer to light it up.

L1 · launchd scheduler
L2 · headless runner · claude -p
L3 · the eight agents
L4 · guards · VERA + PROBE
L5 · approval dashboard · human
L1

Scheduler — launchd

Native macOS jobs fire each agent at its exact minute; missed runs catch up on wake. Survives reboots with no babysitting.

L2

Runner — headless Claude

Each job wakes a fresh Claude with that agent’s instructions. No work product = run reported failed.

L3

The agents

Eight roles across Content, Operations and Quality — each with a prompt, a domain, and a calendar slot.

L4

Guards & tests

VERA screens every post at the publish boundary; PROBE watches health, credentials and sends the weekly report.

L5

Human approval

The dashboard where every post waits for a person. The top layer — nothing ships without it.

Review & validation

The AI proposes. A human disposes.

Autonomy stops at the approval queue. VERA and PROBE make sure only accurate, on-brand, policy-safe content ever gets that far.

👤

Human approval

Every post lands in a queue and waits. The founder reviews image + copy and approves, edits or rejects. Agents never post directly.

Never auto-publish
V

VERA · the guard

A hard screen at the publish boundary blocks any barred topic — on every path, before a single network call.

Enforced in code
P

PROBE · proof of work

A post is only queued once its files truly exist. Agents that produce nothing report failed — no silent false-green.

Self-honest
✉️

External watch

Weekly status emails and daily credential checks alert the team before anything drifts — on-track, failed, or a token about to lapse.

Alerts before it breaks
